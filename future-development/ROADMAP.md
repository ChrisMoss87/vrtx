## Testing & Optimization (Prerequisite)

Before adding new features, the codebase should be fully tested and optimized.

See: [testing-optimization/README.md](./testing-optimization/README.md)

| Initiative | Priority | Details |
|------------|----------|---------|
| Create Missing Factories | High | 18 new factories for models |
| Backend API Tests | High | 150+ endpoints need coverage |
| Frontend E2E Tests | High | DataTable, Forms, Kanban testing |
| Fix N+1 Queries | High | Kanban, Record transform |
| Fix Memory Issues | Medium | Reports, Imports |
| Code Cleanup | Medium | Remove duplicate/unused files |

## Getting Started

1. **First**: Complete Testing & Optimization phase
2. Review feature specifications in each phase directory
3. Prioritize based on customer feedback and business goals
4. Each feature file contains:
   - Overview and user stories
   - Technical requirements
   - Database schema changes
   - API endpoints
   - Frontend components
   - Testing requirements
